# Skill: api-design

API patterns: shaping clear, consistent, evolvable interfaces.

## Competence by weight

- Low (0.0-0.3): functional endpoints, inconsistent shape.
- Mid (0.4-0.6): coherent resources, sane status codes and errors.
- High (0.7-0.9): versioning, pagination, auth patterns, contracts that age well.

## Pairs well with

- `analytical` for systematic interface design.
- `defensive` for input validation and error contracts.
- modular evolution: clean API boundaries are what make a module transplantable in crossover.
